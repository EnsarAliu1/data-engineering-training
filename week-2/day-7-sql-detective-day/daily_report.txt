What SQL mistake I understand better now:
- Misplacing HAVING before GROUP BY. The database must group records before it can filter aggregate values (like counts or sums). Placing HAVING first creates syntax errors because there are no grouped records to check yet.

What was difficult:
- Spotting logical errors that don't trigger syntax warnings. A query like Query 8 (missing JOIN ON) runs without errors but creates a cross join that duplicates rows. You only notice it by carefully double-checking the row count.
