What was difficult:
- The hardest part of this sprint was keeping the sequence of transformations strictly ordered. Since each step relies on the previous one (e.g. we must validate to strip out bad numbers before calculating revenue, and we must normalize statuses before filtering for completed orders), any small shift in function execution order inside main() completely throws off the final output. Building this manually without using Pandas meant keeping close track of dictionary mutations and copying lists explicitly to prevent side effects.

What I understood better today:
- I got a much clearer understanding of why data validation has to happen as the very first step in a data engineering pipeline. Seeing how a single negative quantity (-1) or zero price can skew averages and total revenue numbers made the need for upfront checks very real. I also learned how to use Python's built-in sort and lambda key functions to cleanly rank nested list-of-dictionary structures without needing external tools.

What I need to practice more:
- Managing and refactoring multi-stage dictionary lookups and custom groupings.
- Designing cleaner error handling and typing conventions when working with mixed dictionary keys that might contain None values.
